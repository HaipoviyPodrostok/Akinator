# Акинатор (Akinator)

Консольная реализация игры "Акинатор" на языке C, в которой программа пытается угадать задуманного вами персонажа или предмет, задавая вопросы, на которые можно ответить "да" или "нет". Проект использует Бинарное дерево для хранения базы знаний. Обучение (добавление новых персонажей) происходит прямо во время игры.

## Основной функционал

Программа предоставляет интерактивное меню со следующими возможностями:
1. **Начать новую игру**: Основной режим. Программа задает вопросы, вы отвечаете "да" или "нет". Если программа не угадала, она предложит вам добавить вашего персонажа и его признак, таким образом пополняя базу знаний.
2. **Загрузить сохранение из файла**: Позволяет загрузить базу знаний. Доступен выбор: загрузить стандартную базу (`assets/baze.txt`) или указать путь к своему файлу.
3. **Составить описание объекта**: Выводит полный список признаков выбранного объекта, собранный на основе пути в дереве.
4. **Сохранить дерево в файл**: Сохраняет текущую базу знаний (со всеми новыми добавленными объектами) в текстовый файл для использования в будущем.
5. **Сравнить два объекта**: Показывает общие признаки и различия между двумя объектами на основе их положения в дереве.
6. **Графический дамп дерева**: Генерирует визуальное представление дерева в формате `.dot` и компилирует его с помощью graphviz в `.png` изображение (сохраняется в папку `Graphviz_dumps`).

## Требования для сборки и работы

- Компилятор **gcc**.
- Утилита **Make** для сборки.
- Установленный пакет **Graphviz** для генерации графических графических дампов.

## Сборка и запуск проекта

В корне проекта предусмотрен `Makefile`. Для сборки программы:

```bash
make
```

После компиляции запустите полученный исполняемый файл:

```bash
./a.out
```

> **Примечание:** Перед началом игры рекомендуется загрузить базу данных (Пункт меню **2** -> **1. Загрузить предустановленное дерево**), иначе дерево будет пустым.

## Очистка временных файлов

Для очистки папок с графическими и текстовыми дампами (`Graphviz_dumps` и `Text_dumps`) используйте:

```bash
make clean_dumps
```

## Технические детали
В проекте используется собственная реализация стека (`AkinatorStack`), которая используется для определения пути до объекта при сравнении или описании.
